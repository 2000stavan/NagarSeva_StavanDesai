# NagarSeva (Community Hero)
**Next-Generation AI-Powered Civic Engagement & Municipal Asset Management Platform**

---

## 1. Executive Summary
**NagarSeva (Community Hero)** is a comprehensive, AI-driven civic governance and community resolution platform designed to bridge the gap between citizens, municipal authorities, and on-ground field workers. By combining multimodal artificial intelligence, real-time Indic language translation, GIS heatmap mapping, and gamified civic participation, NagarSeva transforms traditional, opaque grievance redressal into a transparent, efficient, and collaborative ecosystem.

---

## 2. The Problem Statement
Urban municipalities and local governance bodies in rapidly growing urban centers face systemic challenges in civic infrastructure maintenance:
1. **Opaque Grievance Redressal & Citizen Apathy**: Citizens reporting potholes, broken streetlights, water pipe leakages, or uncollected garbage rarely receive actionable updates or proof of resolution. The lack of transparency breeds cynicism and discourages active community participation.
2. **Manual Triage Bottlenecks**: Municipal supervisors receive thousands of unstructured, unverified complaints daily. Manually categorizing issues, assessing hazard severity, estimating repair budgets, and dispatching tasks to appropriate departments is labor-intensive and prone to human error.
3. **Unverified Field Repairs & Quality Control**: Authorities lack reliable, real-time mechanisms to verify if a field worker has executed a repair according to engineering and structural standards. Traditional sign-offs rely on blind trust or delayed physical inspections.
4. **Linguistic Fragmentation**: India’s municipal workforce and citizenry operate across diverse dialects. Language barriers between English/Hindi-speaking administrative authorities and regional field workers frequently lead to miscommunication, delayed execution, and incorrect job completions.
5. **Worker Guidance & Site Safety**: On-ground workers often encounter unexpected site hazards, lack clear step-by-step technical guidance for complex repairs, and struggle with cumbersome manual paperwork.

## 4. Core Artificial Intelligence Innovations

### A. Automated Multimodal Triage (X.AI Grok Vision)
When a citizen uploads an image of an infrastructure failure, the backend analyzes the image using **X.AI Grok Vision (`grok-4.3`)**:
* **Visual Classification**: Automatically categorizes the defect into standardized buckets (`pothole`, `water_leakage`, `streetlight`, `waste`, `road_damage`, `other`).
* **Severity Assessment**: Assigns a risk tier (`low`, `medium`, `high`, `critical`) based on visible hazard indicators (e.g., depth of pothole, volume of water loss, electrical exposure).
* **Automated Cost Estimation**: Generates an immediate estimated financial repair budget in Indian Rupees (`estimated_cost_inr`) along with technical rationale (e.g., *“Requires 2 cubic meters of asphalt mix and labor for compaction”*), giving budget officers instant oversight.

### B. AI Worker Step Verification (Grok Multimodal)
Instead of a generic completion toggle, field workers receive a dynamic, tailored step-by-step repair plan generated by **X.AI Grok Reasoning (`grok-4.20`)**:
* **Mandatory Photo Proofs**: Workers must capture live photo proofs at each step of the assignment (e.g., Step 1: Site clearing, Step 2: Base layer preparation, Step 3: Final compaction).
* **Real-time Quality Inspection**: The multimodal AI inspects the uploaded photo against the step requirements before unlocking the next stage. If a worker attempts to pour asphalt over uncleared debris or flooded pits, the AI flags the quality or safety issue and provides actionable corrective feedback.

### C. Seamless Indic Localization & Voice Help (Sarvam AI)
* **Real-Time Translation Engine**: Integrated directly into the API routing via **Sarvam AI (`sarvam-translate:v1`)**, supporting 11 Indic languages (*Hindi, Marathi, Tamil, Telugu, Bengali, Gujarati, Kannada, Malayalam, Punjabi, Odia, English*).
* **Bidirectional Language Bridging**: Issue descriptions reported by citizens in local languages (e.g., Marathi or Tamil) are automatically translated to English/Hindi for municipal supervisors. Conversely, technical supervisor notes are translated into the worker's native dialect.
* **Native Voice Assistant**: Field workers wearing gloves or operating machinery can speak directly into the app or listen to voice guidance in their regional language to overcome on-site engineering challenges.

## 5. Detailed Portal Workflows & Feature Specifications

### 1. Citizen & School Reporter Portal
* **Interactive GIS Mapping**: View reported community issues on interactive clustering maps. Users can toggle between **Standard View** and **Aging Heatmaps** (powered by `leaflet.heat`) which visually highlight chronic, long-pending infrastructure failures in red/orange gradients.
* **Frictionless Issue Reporting**: Submit detailed civic reports with precise GPS coordinates, photo evidence, and optional whistle-blower anonymity (`is_anonymous`).
* **Civic Gamification & Leaderboard**: Active participation is rewarded. Citizens earn **Civic Score points** and unlock badges (*e.g., First Reporter, Community Guardian, Waste Warrior*) for verified reports, upvotes, and contributing community service hours.
* **Micro-Sponsorships & Crowdfunding**: Local residents, businesses, and NGOs can pledge financial contributions (`sponsored_amount`) directly to specific community repair issues to accelerate priority fulfillment.

### 2. Authority & Supervisor Dashboard
* **360° KPI Analytics**: Comprehensive overview showing open vs. resolved ratios, average resolution timeframe in days, department-wise efficiency breakdowns (*Roads, Electricity, Water, Waste, Parks*), and citizen satisfaction ratings (1–5 stars).
* **Smart Worker Dispatch**: Filter issues by severity or location and assign tasks directly to specialized field workers with scheduled execution dates, priority levels, and estimated work durations.
* **Automated SLA Escalation Engine**: Background workers continuously audit open issues. If an issue remains unresolved beyond defined service-level agreement (SLA) thresholds, automated cron jobs escalate the issue level (`escalation_level: 1, 2, ...`) and dispatch priority alerts to municipal commissioners.
* **Live Unit Tracking & Emergency SOS**: Real-time geolocation tracking of active field workers on an interactive supervisor map. If a worker encounters personal danger or severe site accidents, triggering an **SOS Alert** broadcasts an immediate emergency beacon with precise GPS coordinates to the dashboard.

### 3. Field Worker Mobile Application
* **Optimized Mobile Shell**: Designed specifically for outdoor, on-the-move usage with large touch targets, high-contrast typography, and streamlined workflows.
* **GPS Geofencing Check-in**: Workers verify their presence at the job site through GPS coordinate validation before commencing repair tasks.
* **Materials & Inventory Logging**: Workers record consumed municipal supplies (`materials_used` such as cement bags, asphalt tons, replacement pipes) for accurate municipal inventory auditing.
* **Automated Daily AI Summaries**: At the end of a shift, AI synthesizes all completed jobs, hours logged, and materials consumed into a polished executive summary (`daily_summaries`) sent directly to supervisors.

## 6. System Architecture & Tech Stack

### Frontend Application (`/frontend`)
* **Core Framework**: React 19 bundled with Vite 8 for lightning-fast hot module replacement and optimized production builds.
* **Styling & Design System**: Tailwind CSS v4 utilizing modern utility patterns, glassmorphism headers, responsive grids, and customized color palettes (*Emerald for Citizen, Slate for Authority, Orange for Worker*).
* **GIS & Visualization**: `leaflet` and `react-leaflet` with `leaflet.markercluster` for pin aggregation and `leaflet.heat` for spatial density rendering. `recharts` powers responsive bar charts and KPI metrics on the dashboard.
* **State & Routing**: React Router v7 featuring dedicated Role-Based Access Guards (`<WorkerGuard>`, `<AuthorityGuard>`, `<RoleRedirect>`) to secure multi-tenant navigation.

### Backend API Server (`/backend`)
* **Server Runtime**: Node.js utilizing modern ECMAScript modules (ESM) and Express 4.21.
* **Database Layer**: PostgreSQL database managed via direct, high-performance SQL schemas (`schema.sql`, `worker-schema.sql`) leveraging `pg` connection pooling and `pgcrypto` for native UUID generation.
* **AI & External Integrations**:
  * `openai` SDK pointing to **X.AI Grok API** (`grok-4.3` for multimodal vision inspection, `grok-4.20-0309-reasoning` for step generation and triage).
  * **Sarvam AI REST API** (`sarvam-translate:v1`) with memory-bounded translation caching for sub-second Indic localization.
  * **Cloudinary SDK** for secure image uploads, transformations, and cloud CDN storage.
  * **Nodemailer** for automated SMTP email dispatch during issue reporting and escalations.
* **Background Scheduling**: `node-cron` executing automated daily SLA audits (`startEscalationCron`) and worker summary rollups (`startWorkerCron`).

## 8. Database Schema Overview
The repository maintains a normalized relational structure:
* **`users`**: Stores profile information, hashed passwords (`bcryptjs`), user roles (`citizen`, `authority`, `supervisor`, `worker`), assigned department, and accumulated `civic_score`.
* **`issues`**: Contains title, description, category, severity, status (`open`, `verified`, `in_progress`, `resolved`), latitude/longitude, AI cost estimates, upvotes, and duplicate linkage.
* **`job_assignments`**: Links issues to field workers and supervisors, tracking check-in/check-out timestamps, GPS geofencing verification, and overall job status.
* **`work_steps`**: Stores individual task checkpoints for jobs, recording step photos, AI verification boolean (`ai_verified`), and AI feedback notes.
* **`materials_used`**: Records material quantities and units consumed during field execution.
* **`sos_alerts`**: Emergency records containing coordinates, timestamps, and resolution notes for worker safety incidents.
* **`satisfaction_ratings`**: Citizen feedback ratings (1–5) tied directly to completed issue repairs.

## 9. Local Deployment & Setup Guide

### Prerequisites
* **Node.js** v18+ and **npm**
* **PostgreSQL** instance running locally on port `5432`

### 1. Environment Configuration
Create a `.env` file inside the `backend` directory with the following variables:
```env
DATABASE_URL=postgresql://<your_pg_user>@localhost:5432/community_hero
JWT_SECRET=your_super_secret_jwt_key
PORT=3001
FRONTEND_URL=http://localhost:5173

# AI & Localization Keys (Using lowest cost non-reasoning tier to conserve credits)
XAI_API_KEY=your_xai_grok_api_key
XAI_VISION_MODEL=grok-4.20-0309-non-reasoning
XAI_TEXT_MODEL=grok-4.20-0309-non-reasoning
SARVAM_API_KEY=your_sarvam_indic_api_key

# Optional Media & Email Keys
CLOUDINARY_CLOUD_NAME=your_cloud_name
CLOUDINARY_API_KEY=your_api_key
CLOUDINARY_API_SECRET=your_api_secret
```

### 2. Database Initialization
Run the automated schema setup and seeding scripts inside the `backend` directory:
```bash
cd backend
npm install
npm run setup   # Creates database community_hero and applies schema tables
npm run seed    # Populates mock issues, departments, and demo user accounts
```

### 3. Launching Development Servers
Start the backend server and frontend Vite server in two terminal sessions:
```bash
# Terminal 1: Backend API Server (runs on http://localhost:3001)
cd backend
npm run dev

# Terminal 2: Frontend App (runs on http://localhost:5173)
cd frontend
npm install
npm run dev
```

Open `http://localhost:5173` in your web browser. You will be greeted by the portal login interface where you can sign in using demo accounts for Citizens, Field Workers, or Municipal Authorities!
